# `README` Standards

When writing project `README` files in Motivation projects, it is important to follow the standards and guidelines set below. This allows for clean, concise, and descriptive project `README`s within Motivation.

## Title & Description
Titles should include an optional logo, followed by the repo name with a short description of the project.

```md
<OPTIONAL LOGO>

# `<REPO NAME>`
<REPO DESCRIPTION>
```

Where `<REPO NAME>` is the repository name formatted as code, and `<REPO DESCRIPTION>` is a short description of the repository.

The repository title is the highest level heading, meaning any heading below cannot be a single header (`#`) 

### Example

```md
<img src="resources/logo.png" width="25%">
    
# `System`
System is an example project for complete beginners, allowing you to understand how READMEs should be formatted.  
```

## Ending
Endings should include information about the repository's license and open source promise.

```md
## Free & Open-Source

<REPO NAME> is 100% free with no drawbacks or limitations. There is no "premium" version; you get the latest and greatest, all licensed under the <REPO LICENSE>.

All source code is public, to anyone. There is no "hidden mechanism" included in this repository; every reference and used factor exists completely and fully.
```

Where `<REPO NAME>` is the repository name, and `<REPO LICENSE>` is an abbreviation of the repository license (e.g. `GPL-3.0`)
